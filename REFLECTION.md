# Reflection on GitHub Kanban Implementation

## Challenges
- Template selection: Basic templates lacked automation; Automated Kanban required learning workflow triggers.
- Customization: Adding "Testing"/"Blocked" columns needed precise field configs to integrate with issues.
- Linking Assignment 6: Mapping sprint tasks (e.g., 10 user stories) to issues took iteration for labels/assignments.

## Comparison to Other Tools
| Tool | Pros | Cons vs GitHub |
|------|------|---------------|
| GitHub Projects | Native GitHub integration, free, automation | Less visual polish |
| Trello | Drag-drop ease, power-ups | No Git issue linking |
| Jira | Advanced reporting, WIP enforcement | Costly, complex setup |

GitHub wins for our repo-integrated workflow. Lessons: Start with automation-rich templates; define custom columns early for Agile flow.
