# Reflection on State and Activity Modeling

## Challenges
- Granularity: States/actions balanced for readability (e.g., avoided micro-states in Booking).
- Agile Alignment: Diagrams directly from user stories (Ass6), e.g., "As student, book room" → Resource/Booking states.

## Comparison
- **State Diagrams**: Object-centric (internal behavior, e.g., Resource states).
- **Activity Diagrams**: Process-centric (interactions/swimlanes, e.g., booking workflow).

State diagrams detail lifecycles; activities show flows. Together, prepare for implementation (e.g., DB schemas from states).
