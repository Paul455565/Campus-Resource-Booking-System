## Stakeholder Analysis Table

| Stakeholder | Role | Key Concerns | Pain Points | Success Metrics |
|-------------|------|--------------|-------------|-----------------|
| **Students** | Primary users who book resources (study rooms, projectors, labs) for academic work. | Easy access to availability, quick booking, real-time status updates, mobile-friendly interface. | Manual sign-ups lead to double-bookings; no visibility into availability; forgotten bookings cause no-shows. | 90% booking success rate within 3 clicks; average booking time <1 min; <5% cancellation rate. |
| **Faculty/Teachers** | Book resources for classes/tutorials; sometimes oversee student bookings. | Reliable scheduling for teaching sessions; priority access during peak hours; integration with class rosters. | Conflicts with student bookings; delays in approvals disrupt lesson plans. | 95% approval rate for faculty bookings within 2 hours; zero double-bookings for classes. |
| **Resource Administrators** | Manage daily operations: approve/reject bookings, handle queries, update resource status. | Streamlined approval workflow; automated reminders; overview dashboards for occupancy. | Overloaded with emails/phone calls; manual tracking of resource maintenance. | Reduce approval time by 50% (target: <30 min/booking); 20% fewer support queries. |
| **IT Support Staff** | Maintain system uptime, handle technical issues, manage user accounts. | System reliability, easy monitoring, low maintenance overhead; security compliance. | Frequent crashes during peak hours; debugging complex manual processes. | 99.5% uptime; <2h resolution for incidents; support tickets reduced by 40%. |
| **University Administrators** | Oversee resource utilization, budgeting, policy enforcement. | Fair allocation across departments; usage analytics for decision-making; cost efficiency. | Lack of data on under/over-utilized resources; disputes over fairness. | Resource utilization >75%; generate monthly reports; 100% policy compliance (e.g., no unauthorized bookings). |
| **Maintenance Staff** | Schedule resource maintenance/downtime; update availability. | Simple interface to mark resources unavailable; notifications for upcoming maintenance. | No system to flag downtime, leading to invalid bookings. | 100% bookings avoided during maintenance; <10 min to update status. |
| **Security Auditors** | Ensure data privacy, access controls, audit trails. | Comprehensive logging, encryption, role-based access; compliance with regulations (e.g., GDPR-like). | No audit trails for bookings; risk of unauthorized access. | Full audit logs available; zero security incidents; annual compliance certification. |

**Traceability:** Concerns directly map to FR (e.g., booking UI for students) and NFR (e.g., uptime for IT) in SYSTEM_REQUIREMENTS.md. Analysis ensures balanced priorities.

**Total Stakeholders:** 7 (exceeds requirement).
