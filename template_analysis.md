# GitHub Project Templates Analysis and Selection

## Comparison Table

| Template | Columns/Workflow | Automation Features | Suitability for Agile |
|----------|------------------|---------------------|----------------------|
| Basic Kanban | To Do, In Progress, Done | None | Basic task tracking; limited for sprints |
| Automated Kanban | To Do, In Progress, Review, Done | Auto-move on labels/pull requests, status updates | High: Supports sprint tracking, automation for CI/CD |
| Bug Triage | Triage, Confirmed, In Progress, Done | Auto-assign on labels, priority sorting | Good for maintenance; less for feature dev |
| Team Planning | Now, Next, Later + Time-based | Roadmap views, iteration planning | Excellent for roadmaps; Agile planning |

## Justification
Selected **Kanban Board** (as per your choice) for the Campus Resource Booking System project. It provides essential Agile workflow:
- Simple columns for To Do → In Progress → Done, customizable for sprints.
- Supports task tracking from Assignment 6 (user stories like booking UI, backend API).
- Team-friendly with @mentions, labels (bug, feature), and easy customization (add Testing/Blocked).
This template fits our iterative dev perfectly (frontend booking, resource DB, auth).
